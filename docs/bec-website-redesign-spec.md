# BEC Solutions Website Redesign Spec

## Purpose

This document is the source of truth for redesigning the BEC Solutions website.

The goal is to transform the current site from a minimal, generic business page into a premium, modern, trustworthy small-business website focused on practical workflow automation and process improvement.

This should guide all design, copy, layout, component, and implementation decisions.

---

## Primary Goal

Create a homepage and supporting site structure that makes BEC Solutions feel:

* practical
* modern
* premium
* trustworthy
* operationally sharp
* SMB-friendly

The site should clearly communicate that BEC Solutions helps small businesses reduce manual work, improve response times, and streamline operations using practical automations that fit into existing workflows.

---

## Design Direction

### Desired look and feel

The site should feel:

* premium
* modern
* sharp
* structured
* calm
* confident
* trustworthy
* practical

### Avoid

Do not make the site feel:

* gimmicky
* over-animated
* experimental
* edgy for the sake of it
* like a generic startup template
* like a flashy AI demo page
* like a chatbot agency

### Visual style direction

Use:

* dark charcoal hero section
* strong typography hierarchy
* lots of whitespace
* premium card layouts
* restrained accent color
* subtle motion only
* slight bento-grid influence in service and use-case layouts
* crisp section contrast
* clean icons
* slightly asymmetric or organic layout touches in moderation

Do not use:

* heavy gradients everywhere
* neon tech styling
* loud 3D visuals
* neo-brutalist styling
* too many decorative animations
* tiny low-contrast text

---

## Design Trends to Apply

Based on the design direction selected, these are approved:

### Use these

* subtle micro-interactions
* strong/bold typography
* dark/light contrast
* accessibility-first design
* performance-conscious design
* mild bento-grid layout
* subtle asymmetry in select sections

### Use sparingly

* kinetic text only in the hero, if at all
* organic shapes only as a subtle design accent

### Do not prioritize

* AI personalization
* heavy 3D visuals
* anti-design / neo-brutalism
* aggressive experimental layouts

### Mobile-first

All sections must be designed mobile-first.
The site should feel strong on phone screens, since many SMB owners will likely view it on mobile.

### Accessibility

Ensure:

* strong contrast
* visible focus states
* labeled form fields
* keyboard navigability
* large enough tap targets
* restrained motion
* semantic heading structure

### Performance

Keep the site light.
Avoid unnecessary dependencies, overbuilt animations, and oversized assets.

## Implementation Notes for Cursor

### Key implementation priorities

1. Improve the visual polish and modernity of the homepage
2. Preserve clarity and trust over flashy effects
3. Build reusable components and avoid one-off messy code
4. Keep section spacing and typography consistent
5. Maintain mobile responsiveness throughout
6. Optimize for performance and readability

### Code quality expectations

* keep components modular
* avoid unnecessary complexity
* preserve clean folder structure
* avoid giant monolithic page files where possible
* use reusable section components if appropriate
* ensure semantic HTML and accessible structure

### If animations are added

* use subtle fade/translate/hover effects only
* do not over-animate text or entire layouts
* keep transitions smooth and restrained

### If icons are used

* keep them consistent in size and style
* prefer simple, clean iconography

---

## What Cursor Should Not Do

* do not rewrite the brand into a generic AI startup
* do not introduce excessive buzzwords
* do not make the site look like a SaaS dashboard product
* do not add flashy gimmicks just because they are trendy
* do not overload the homepage with too much visible text at once
* do not weaken clarity in pursuit of style
* do not force dashboards or apps into the messaging unless clearly useful
* do not add visual clutter

---

## Suggested Working Method in Cursor

Cursor should approach this in phases:

### Phase 1: Audit

* review the current project structure
* identify homepage entry points, layout files, components, styles, and design system constraints
* summarize how the current site is organized before major edits

### Phase 2: Planning

* propose a component map for the new homepage
* identify which components can be reused, modified, or replaced
* confirm section order and implementation plan

### Phase 3: Homepage redesign

* implement the new homepage structure
* update copy to match this spec
* redesign hero, services, pain points, process, use cases, CTA sections

### Phase 4: Polish

* improve spacing, responsiveness, typography, hover states, and visual hierarchy
* refine accessibility and performance

### Phase 5: Final QA

* check for mobile issues
* check CTA consistency
* check heading structure
* check contrast and readability
* check for generic or off-brand language
